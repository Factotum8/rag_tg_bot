# RAG bot

## Исследование моделей и инфраструктуры

### Сравните LLM-модели (локальные Hugging Face vs облачные OpenAI)

Для анализа были выбраны две модели из разных классов:
Hugging Face / Mistral-7B — локальная модель, которую можно развернуть on-premises.
OpenAI / gpt-5.1-mini — облачная модель, представляющая собой де-факто стандарт среди коммерческих LLM-провайдеров.
Это два принципиально разных подхода к использованию LLM, поэтому рассматривать только один из них было бы недостаточно.


| Метрика                    | Hugging Face (Mistral-7B)                                | OpenAI (gpt-5.1-mini)                        |
| -------------------------- | -------------------------------------------------------- |----------------------------------------------|
| **Качество ответов**       | – Низкое «из коробки», требует хорошего контекста (RAG). | + Выше по умолчанию.                         |
| **Скорость работы**        | – Ниже на Mac M4 Pro (48 GB).                            | + Существенно быстрее на Mac M4 Pro (48 GB). |
| **Стоимость владения**     | + Бесплатные запросы, но требуется аренда/поддержка GPU. | + Нет своих серверов, но запросы платные.    |
| **Простота развёртывания** | + Легко поднять локально в on-prem среде.                | + Вообще не требует развёртывания.           |

Тестовый код моделей:
[gpt-5-mini.py](llms/gpt-5-mini.py)
[mistralai.py](llms/mistralai.py)

Проверочные вопросы для оценки качества ответов. Для тестирования использовался единый набор вопросов:
* Объясни простыми словами, что такое RAG.
* SaaS-платформа для моделирования промышленных объектов «Digital Twin». Расскажи историю создания таких систем.
* Что такое цифровые двойники в SCADA-системах?

Примеры ответов моделей:
[reply_example_gpt_mini.txt](reply_example_gpt_mini.txt)
[reply_example_mistralai.txt](reply_example_mistralai.txt)

Итоги сравнения:
По сумме критериев лучше показал себя OpenAI, поскольку предоставляет полностью готовую инфраструктуру и обеспечивает более высокое качество ответов без дополнительной настройки.
Тем не менее для реализации чат-бота была выбрана Mistral-7B, так как она подходит для on-prem развертывания, что является критичным требованием для ИБ при использовании корпоративной базы знаний в качестве контекста для LLM.


### Сравните модели эмбеддингов 

| Метрика                                | Sentence-Transformers                                                                                                                                     | OpenAI Embeddings                                                                                                                                                                                  |
|----------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **скорость создания индекса**          | + <br/>Быстро, если модель маленькая (all-MiniLM-L6-v2) и есть CPU/GPU. <br/>Можно параллелить в несколько потоков. <br/>Нет задержек сети, всё локально. | + Очень высокая производительность — расчёт идет на серверных GPU. <br/>Можно отправлять большие батчи (до ~2k токенов на запрос). <br/>Можно параллелить запросы → практически линейный скейлинг. |
| **качество поиска**                    | – Ниже на Mac M4 Pro (48 GB).                                                                                                                             | + Существенно быстрее на Mac M4 Pro (48 GB).                                                                                                                                                       |
| **стоимость владения и использования** | + Бесплатные запросы, но требуется аренда/поддержка GPU.                                                                                                  | + Нет своих серверов, но запросы платные.                                                                                                                                                          |

Итог:
OpenAI embeddings обычно дают лучшее усреднённое качество на разных доменах (код, Wikipedia, вопрос-ответ, мультиязычность).
Sentence-Transformers может выигрывать, если используется специализированная модель, обученная под конкретную задачу.

### Сравните векторные базы ChromaDB и FAISS

| Метрика                                      | ChromaDB                                                                                             | FAISS                                                                                            |
|----------------------------------------------|------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------|
| **скорость поиска и индексации**             | - На миллионах векторов — медленнее FAISS. Индексация не требует ручной настройки.                   | + Индексация требует ручного выбора типа индекса. Один из самых быстрых ANN-движков в мире.      |
| **сложность внедрения и поддержки**          | + Хранит метаданные. Сохраняет коллекции (persist). Обеспечивает фильтрацию.                         | - Встроенной БД нет.                                                                             |
| **удобство в работе**                        | + Приятный API: add(), query(), update(), delete(). Автоматическое хранение документов и метаданных. | - Нет встроенной поддержки документов, текстов, метаданных. Нет фильтрации по полям.             |
| **стоимость владения (учёт инфраструктуры)** | - Прост на старте - сложен при высокой нагрузке. Нужно масштабировать вручную.                       | - Бесплатен. Администрирование на вашей стороне. Сложен на старте - сложен при высокой нагрузке. |

Итог:
FAISS — мощный движок, но нужна инфраструктура вокруг него.
ChromaDB быстрее FAISS на старте (zero-config), но медленнее на больших данных. 
ChromaDB — намного проще FAISS в интеграции и эксплуатации.
Chroma — гораздо удобнее в реальной разработке и RAG.
Chroma - дёшево в использовании для малых и средних систем, дороже для крупных. 
Согласно статистике из ТЗ мы можем смело использовать ChromaDB так как коллекция документов ~21250 документов. 

### Конфигурацию сервера

#### Вариант 1 — Полностью on-prem: Mistral-7B + ChromaDB + Sentence-Transformers

##### Инструменты

* LLM: **Mistral-7B-Instruct** (Hugging Face, quantized 4–8 bit).
* Эмбеддинги: **Sentence-Transformers** (`all-MiniLM-L6-v2`).
* Векторная БД: **ChromaDB**.
* API-слой: **FastAPI** / **gRPC**.
* Оркестрация: Docker / Kubernetes (по желанию).

##### Рекомендуемая конфигурация сервера

**Базовый прод-сервер под одну модель 7B:**

* **CPU:** 16 vCPU (8–16 физических ядер, например Xeon / Ryzen / EPYC).
* **RAM:** 64 GB.
* **GPU:**

  * NVIDIA с **24 GB VRAM** (например RTX 4090 / A5000 / L40 и т.п.)
  * Либо 2 × 16 GB (но лучше одна 24).
* **Диск:**

  * 1–2 TB NVMe SSD (модель + индексы + логи).
* **Сеть:** 1 Gbit/s внутри кластера, доступ в интернет опционален (для обновлений).

Такой сервер позволить:

* держать quantized Mistral-7B целиком в GPU,
* обрабатывать десятки запросов в минуту с нормальной задержкой,
* хранить векторный индекс (Chroma/FAISS) локально.

##### Плюсы

* Полный контроль над данными (Хранилище, LLM, индексы — всё внутри периметра).
* Можно пройти жёсткие требования ИБ.
* Нет зависимости от внешнего API (квоты, деньги, блокировки).
* Масштабируемо: можно добавить ещё GPU/серверы.

##### Минусы

* Самая высокая стоимость входа (сервер с GPU).
* Нужна MLOps/DevOps-компетенция (обновления, мониторинг, бэкапы).
* Качество ответов Mistral-7B чуть ниже топовых облачных LLM (OpenAI gpt-5.1).

#### Вариант 2 — Гибрид: локальные эмбеддинги + ChromaDB, LLM в облаке (OpenAI)

##### Инструменты

* Эмбеддинги: локально (Sentence-Transformers) или через OpenAI Embeddings.
* Векторная БД: **ChromaDB** on-prem.
* LLM: **OpenAI (gpt-5.1-mini / gpt-5.1)** по API.
* API-слой: FastAPI (сервер внутри компании, который общается с OpenAI).

##### Рекомендуемая конфигурация сервера (без локального LLM)

* **CPU:** 8 vCPU.
* **RAM:** 32 GB.
* **GPU:** не обязателен (можно считать эмбеддинги на CPU или брать OpenAI Embeddings).
* **Диск:** 500 GB–1 TB SSD (векторный индекс + документы).

##### Плюсы

* Нет затрат на GPU → дешевле железо.
* Отличное качество ответов (OpenAI LLM сильнее Mistral-7B).
* Локальная векторная база — база знаний не уходит целиком в облако, только куски контекста.

##### Минусы

* Всё равно отправляются части текста (контекст) в облако → вопрос ИБ.
* Зависимость от внешнего API (лимиты, цена, доступность, политика).
* Для строго секретных данных этот вариант может не подходить по требованиям ИБ.

#### Вариант 3 — Полностью облачное решение

##### Инструменты

* LLM: **OpenAI gpt-5.1 / gpt-4o**.
* Эмбеддинги: **OpenAI Embeddings** (`text-embedding-3-*`).
* Векторная БД: облачная Qdrant Cloud.
* Ваш сервер: только тонкий backend / frontend.

##### Рекомендуемая конфигурация (тонкий gateway-сервер)

* **CPU:** 2–4 vCPU.
* **RAM:** 8–16 GB.
* **GPU:** не нужен.
* **Диск:** 100–200 GB достаточно.

##### Плюсы

* Минимальная инфраструктура — всё сложное «в облаке».
* Быстрый старт, время до MVP минимальное.
* Масштабирование по сути на стороне провайдеров.

##### Минусы

* Максимальные риски с точки зрения ИБ (все данные идут в облако).
* Сильно завязаны на коммерческую модель провайдеров (цены, квоты).
* Может быть нельзя юридически (информация ограниченного доступа и т.д.).

## Вариант 4 — Компромисс для пилота: on-prem индекс + маленькая облачная модель

Это промежуточный путь для пилота/PoC:

* Векторная БД: **ChromaDB** on-prem.
* Эмбеддинги: OpenAI (для экономии времени и кода).
* LLM: **gpt-5.1-mini** (дёшево, быстро).
* Сервер: как в варианте 2 (8 vCPU, 32 RAM, без GPU).

Используется как временное решение, пока команда не поднимет свой on-prem LLM.

## Рекомендация для «кейса компании»

Контекст:
* база знаний (SCADA, Digital Twin, промышленные объекты) — чувствительная,
* OpenAI хорош по качеству, но **передача KB наружу — критична**,

**Стратегия**

> * **1 этап (3–6 месяцев):**

  * Вариант 4 (гибрид/облак) → быстрый прототип с OpenAI.
  * Проверка UX, бизнес-ценности, сценариев.

> * **2 этап (6–12 месяцев):**

  * Переезд на **вариант 1** (on-prem Mistral-7B + ChromaDB).
  * Интеграция с IDM/AD, логирование, мониторинг.
